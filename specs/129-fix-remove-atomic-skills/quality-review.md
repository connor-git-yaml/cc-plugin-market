## WARNING 建议

### W-01：三个 SKILL.md 的 prompt 来源映射死代码路径（可维护性）

**位置**：
- `plugins/spec-driver/skills/spec-driver-implement/SKILL.md` L123-130
- `plugins/spec-driver/skills/spec-driver-story/SKILL.md` L114-121
- `plugins/spec-driver/skills/spec-driver-resume/SKILL.md` L99-106

**描述**：三个 SKILL.md 的 prompt 来源映射逻辑中，仍包含"检查 `.claude/commands/spec-driver.{phase}.md` 是否存在"的分支（前两个优先分支）。删除原子文件后，这两个分支在本仓库永远不会命中，成为死代码路径。当前行为通过 fallthrough 到 `$PLUGIN_DIR/agents/{phase}.md` 仍正确，但残留逻辑会使维护者产生疑问（"原子文件路径还会被读吗？"）并与 stale reference 清理目标形成矛盾。

**修复建议**：在后续清理 PR 中，将 `prompt_source` 映射简化为：

```text
对于 phase ∈ [specify, clarify, plan, tasks, analyze, implement]:
  if 当前运行时为 Codex 且 .codex/commands/spec-driver.{phase}.md 存在:
    prompt_source[phase] = ".codex/commands/spec-driver.{phase}.md"
  else:
    prompt_source[phase] = "$PLUGIN_DIR/agents/{phase}.md"
```

或在注释中明确说明前两个分支仅用于第三方项目自定义覆盖场景，本仓库不再使用。

**本次影响**：不阻断当前 fix，建议作为后续独立 INFO 级清理任务。

### W-02：`check-prerequisites.sh` L118 错误提示命令语义欠精确（可读性）

**位置**：`.specify/scripts/bash/check-prerequisites.sh` L118

**描述**：当 `tasks.md` 不存在时，提示 `Run /spec-driver:spec-driver-implement first to create the task list`。但 `spec-driver-implement`（无 `--entry-point`）会触发整个 implement 工作流（plan → tasks → implement → verify），而非仅生成 tasks.md。更精确的建议是 `--entry-point=tasks`（如果编排器支持），或说明需要先确保 spec.md 和 plan.md 存在再运行完整 implement 流程。

**修复建议**：

```bash
echo "Run /spec-driver:spec-driver-implement --entry-point=tasks first to create the task list." >&2
```

或根据 `spec-driver-implement` 是否实际支持 `--entry-point=tasks` 决定措辞。

**本次影响**：功能正确（用户按提示运行 implement 会最终生成 tasks.md），仅为用户体验精度问题。

### W-03：`CHANGELOG.md` 格式偏离 Keep a Changelog 标准（可维护性）

**位置**：`CHANGELOG.md` L1-31（整个文件）

**描述**：CHANGELOG.md 使用 `### spec-driver — BREAKING` 作为子节名，而标准 Keep a Changelog 格式要求使用 `### Added`、`### Removed`、`### Changed`、`### Deprecated`、`### Fixed`、`### Security` 等预定义类别。当前格式对工具链解析（如自动生成 release notes 的工具）可能不友好。

**修复建议**：按 Keep a Changelog 格式重构：

```markdown
## [Unreleased]

### Removed

- **[BREAKING]** 删除 9 个遗留原子命令 `/spec-driver.{specify,...}`...
```

**本次影响**：文档可读性良好，仅为格式规范问题，不影响功能。
