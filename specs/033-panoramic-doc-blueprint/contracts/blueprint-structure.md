## Feature 卡片标准格式

每个 Feature 在第 4 章中按以下固定格式呈现：

```markdown
#### Feature {NNN}: {Feature 名称}

**描述**: {一句话描述}
**所属 Phase**: Phase {N}（{Phase 名称}）
**预估工作量**: {N-M 天}
**前置依赖**: {依赖列表，格式为 "Feature {NNN}（强）/ Feature {NNN}（弱）"，无依赖时写"无"}

**交付物**:
- {交付物 1}
- {交付物 2}

**验证标准**:
1. {可观测的预期结果 1}
2. {可观测的预期结果 2}
```

### 字段约束

| 字段 | 必填 | 格式约束 |
|------|------|---------|
| Feature 编号 | 是 | 034-050，三位数字 |
| Feature 名称 | 是 | 中文名称 |
| 描述 | 是 | 一句话，不超过 50 字 |
| 所属 Phase | 是 | Phase 0/1/2/3 |
| 预估工作量 | 是 | "N-M 天" 格式（如 "1-2 天"） |
| 前置依赖 | 是 | Feature 编号列表 + 强/弱标记 |
| 交付物 | 是 | 至少 1 项 |
| 验证标准 | 是 | 至少 2 条 |
